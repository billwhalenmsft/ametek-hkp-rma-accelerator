### What it does
Adds a new **Body Preview** tab to the email log form that renders the inbound email's

back to escaped plain text when `rma_body` is not HTML.

### Verify

**`Body Preview`** ← new. Click it to see the email body inline.

### Three rules required (codified into user memory)
1. Web resource creation **must include `languagecode = 1033`** — empty causes silent dependency lookup failures (`PrimaryNameLookup error 0x8004f036`)

3. Form XML must include `<WebResourceId>{guid}</WebResourceId>` parameter for dependency resolution

## 4. Help subarea + cheat sheet

**Web resource id:** `88feaef7-2950-f111-a824-0022480a5e8d`  
**Web resource name:** `rma_/help/help.html`  
**Sitemap id patched:** `2191c458-1f4e-f111-bec6-000d3a5aed87`  
**Backup:** `customers/ametek/hkp_rma/backup/rma_sitemap_*_PRE_HELP.xml`  
**Script:** `customers/ametek/hkp_rma/scripts/deploy_help_subarea.ps1`  
**Source HTML:** `customers/ametek/hkp_rma/ui/rma_help.html`

### What it does
Adds a **Help** group at the bottom of the left sitemap with a single **Help & Cheat Sheet**
subarea. Opens an embedded styled cheat sheet covering: filing claims, processing
inbound email, reading Smart Insights, requesting approval, stage progression, finding
prior claims, saved views, settings tables, and troubleshooting.

### Verify
Open the app → look for `Help` group at the bottom of the left sitemap →
click `Help & Cheat Sheet` → in-app help page renders.

### Rules learned (different from form patches!)
- Sitemap SubArea reference uses `Url="$webresource:rma_/path/file.html"` — **REQUIRES** the `$webresource:` prefix (opposite of form XML rule!)
- No `Type="Url"` attribute — sitemap XSD rejects it; URL alone is the indicator
- No `<Titles>` / `<Descriptions>` / `<DependencyData>` blocks — uses inline `Title="..."` attribute on SubArea (sitemap XSD differs from older sitemap.xml file format)
- No `ShowGroupName` attribute on Group — XSD rejects it
- **Read existing SubArea entries in the same sitemap and copy their schema verbatim** — saves a half-hour of XSD validation iteration

### Rollback
Remove the `<Group Id="rma_group_help">...</Group>` from the sitemap XML and republish:
```powershell
$o="https://org6feab6b5.crm.dynamics.com"; $t=az account get-access-token --resource $o --query accessToken -o tsv
$bk = "customers/ametek/hkp_rma/backup/rma_sitemap_*_PRE_HELP.xml"
$xml = [IO.File]::ReadAllText((Resolve-Path $bk | Select-Object -Last 1).Path)
Invoke-RestMethod -Method Patch -Uri "$o/api/data/v9.2/sitemaps(2191c458-1f4e-f111-bec6-000d3a5aed87)" -Headers @{Authorization="Bearer $t";"Content-Type"="application/json"} -Body (@{sitemapxml=$xml} | ConvertTo-Json)
```
