## 🎁 What I built — the new dashboard

**Open it now to preview:**
```
customers/ametek/hkp_rma/ui/rma_claims_dashboard.html
```
Just double-click the file. It loads with mock RMA data and works fully (drag cards
between columns, click cards, filter, search). When embedded in the MDA, the same
file detects `Xrm.WebApi` and switches to live Dataverse queries on `rma_claim`.

### Features (matches/extends the CodeApp)
- ✅ **KPI cards**: Open Claims, Avg Age, Approval Needed (>$500 in Decision), Closed This Week
- ✅ **Drag-n-drop Kanban** with 5 status columns (New → Triage → Investigation → Decision → Closed)
  - Drops trigger optimistic UI update, then `Xrm.WebApi.updateRecord` PATCH on `rma_status`
  - Failed updates auto-revert with toast notification
- ✅ **Pipeline ↔ List view** toggle (matches CodeApp UX)
- ✅ **Awaiting Your Action** amber banner for claims with `rma_haspendingresponse=true`
- ✅ **Filters**: search box (claim# / customer / part), plant dropdown, status dropdown
- ✅ **Click claim card** → opens MDA form via `Xrm.Navigation.openForm` (works in MDA), or no-op toast in standalone preview
- ✅ **New Claim button** → opens blank rma_claim form in MDA
- ✅ **Light/dark theme** auto via `prefers-color-scheme`
- ✅ **Status banner** in top-right: 🟢 Live (N claims) when in MDA, 🟡 Preview mode when standalone
- ✅ **Auto refresh button**

### Dependencies (CDN — work in MDA web resources unless tenant CSP blocks)
- TailwindCSS Play CDN
- SortableJS for drag-n-drop
- Inter font from Google Fonts

If your tenant blocks external scripts in web resources, I can rebuild with everything
inlined — about 20 min of work.

## 🚀 To deploy it as the actual Claims Board (when you're ready)

```powershell
powershell -ExecutionPolicy Bypass -File "customers\ametek\hkp_rma\scripts\deploy_dashboard_webresource.ps1" -Publish
```

Then in maker portal:
1. Solutions → open RMA Returns Monitor solution → Add existing → Web Resource → `cr74e_/rma/dashboard.html`
2. Apps → edit "RMA Operations and Monitoring" → Sitemap → add Sub Area:
   - **Type**: URL
   - **URL**: `$webresource:cr74e_/rma/dashboard.html`
   - **Title**: Claims Board
   - **Icon**: any
3. Save & publish app
4. Refresh — click "Claims Board" in left nav → see the dashboard

To roll back: just remove the sub-area from the sitemap. The web resource stays available.
