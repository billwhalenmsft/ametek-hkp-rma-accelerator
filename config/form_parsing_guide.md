# HKP RMA Intake Form — Parsing Guide

> **Purpose:** Reference for the HKP RMA intake agent to correctly parse inbound emails
> generated by the haydonkerkpittman.com website "Quality Support" contact form.
> Updated: 2026-05-11 — calibrated from real Roche Diagnostics submission.

---

## Email Format

Submissions arrive as plain-text emails. The header line is:

```
{Contact Name}, your form has been submitted. Our representatives will contact you.

The information below was summited using the Quality Support form found on www.haydonkerkpittman.com.
```

All fields follow as `Label: Value` lines. Example:

```
Company: Roche Diagnostics International AG
Email: Rath.Feil@roche.com
Phone: +41 - 41 - 792 3660
Address returned parts will ship to:  Rotkreuz, Zug 6343 Switzerland
Quantity of Suspect Parts: 13
PO Number parts were ordered on: 7100328936
Manufacturing Date Code/Serial Number, please see label on parts: P2550-00xxx and P2608-00xxx
AMETEK Part Number: LC1574W-05-B25
MFG Location: Penang, MY
What is the complaint reason? With details (Please send photos/videos): Quality Complaint
What is the complaint reason - Other: [detailed narrative]
Please select your primary HKP sales representative: Istvan Nagy
How was the suspect part detected: Check failed: Motor does not move [4 pieces] and Motor move with low force [9 pieces]
Product Description: 09287868001 STEPPER MOTOR LC1574W-05 13mm PPS Nut
Where in your process was the suspect part detected: Test before use in the production line
Is there a Non-Conformance Report? What is the Non-Conformance Number: [blank or NCR#]
Other Comments or explanation of your concern: [optional]
```

## Field Mapping → Dataverse Entity (`cr74e_warrantyclaim`)

| Form Field | Dataverse Field | Notes |
|---|---|---|
| Company | `customerid` (account lookup) | Match against account name; create contact if not found |
| Email | `emailaddress` | Primary contact |
| Phone | `telephone1` | Optional |
| Address returned parts will ship to | `cr74e_returnaddress` | Shipping address for label |
| Quantity of Suspect Parts | `cr74e_quantitysuspect` | Integer; triggers batch rule if ≥ 5 |
| PO Number parts were ordered on | `cr74e_ponumber` | Used for shipment lookup |
| Manufacturing Date Code/Serial Number | `cr74e_serialnumber` | May contain range (P2550-P2608) |
| AMETEK Part Number | `cr74e_partsku` | Join to `cr74e_parts` |
| MFG Location | `cr74e_mfglocation` | Free text — flag if Penang + batch |
| What is the complaint reason? | `cr74e_complaintreason` | Enum: Quality Complaint, Damaged, Other |
| What is the complaint reason - Other | `description` | Free text narrative |
| Please select your primary HKP sales representative | `cr74e_salesrep` | Name lookup |
| How was the suspect part detected | `cr74e_detectionmethod` | Free text — parse failure counts |
| Product Description | `cr74e_productdescription` | Including product number prefix |
| Where in your process was the suspect part detected | `cr74e_detectionstage` | Process stage |
| Is there a Non-Conformance Report? | `cr74e_ncrnumber` | Nullable |
| Other Comments | `cr74e_additionalcomments` | Optional context |

## Intake Agent Parsing Rules

### 1. Batch Detection
- If `Quantity of Suspect Parts` ≥ 5 → set flag `isBatchClaim = true`
- Auto-escalate to Quality Engineer when `isBatchClaim = true` AND customer is medical/pharma OEM

### 2. Failure Mode Extraction from "How was the suspect part detected"
Pattern: `Check failed: {symptom1} [{count1} pieces] and {symptom2} [{count2} pieces]`

Extract as structured list:
```json
[
  { "count": 4, "symptom": "Motor does not move" },
  { "count": 9, "symptom": "Motor move with low force" }
]
```

### 3. Serial / Date Code Range Handling
- Pattern: `P2550-00xxx and P2608-00xxx` = two distinct date code ranges
- Store as comma-separated in `cr74e_serialnumber`
- Flag for batch investigation when ≥2 date code ranges

### 4. Warranty Eligibility
- LC-series motors (prefix `LC`): check `warrantyMonths` from product master
- No serial-level data (only date code ranges) → use PO date + standard warranty period
- If detection was "Test before use in production" → strong eligibility signal (no customer misuse)

### 5. Priority Scoring
| Condition | Priority |
|---|---|
| Medical/pharma OEM customer | High |
| Quantity ≥ 10 | High |
| Multiple failure modes in one batch | High |
| Mfg location flag (Penang known issue run) | High |
| Detection = pre-production test | Medium (not field failure) |

### 6. Auto-Escalation Triggers
- Batch ≥ 5 units
- Medical/pharma/diagnostics OEM
- Same MFG location + date code appears in >1 claim in 30 days → batch investigation flag

## Notes for CS Agent Topic Design

- The email will NOT have a meaningful subject line — use `Complaint reason - Other` as description
- The contact name in the header (`{Name}, your form has been submitted`) is the submitter, not always the company
- NCR field is frequently blank — treat as nullable, don't fail parsing
- Sales rep field maps to internal rep lookup; if not found, assign to default intake queue
- All timestamps should be interpreted as UTC unless phone country code implies otherwise
