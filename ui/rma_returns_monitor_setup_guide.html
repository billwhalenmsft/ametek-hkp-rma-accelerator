1. What you're getting

The RMA Returns Monitor is a self-contained Power Apps Canvas application designed to streamline your Return Material Authorization (RMA) intake and triage workflow. It captures inbound RMA requests from customers, routes them to the right manufacturing plant, and tracks them through the full claim lifecycle.

Components inside the package

ComponentTypePurpose
RMA Returns MonitorCanvas AppThe main user interface for managing returns
RMA ClaimDataverse TableMaster record for each return request
PlantDataverse TableManufacturing locations that process returns
Routing RuleDataverse TableConfigurable plant assignment logic
Claim NoteDataverse TableInvestigation history & activity log
Approval RecordDataverse TableAudit trail for credits exceeding thresholds
Self-contained design: This solution does not require Microsoft Field Service, Customer Service, or any other Dataverse application. All required tables are included.

3. Installation — step by step

1

Sign in to Power Apps

  1. Open https://make.powerapps.com in your browser
  2. Sign in with your work account
  3. In the top-right environment picker, switch to the target environment where you want the app installed
2

Open the Solutions area

  1. In the left navigation, click Solutions
  2. If you don't see Solutions, click ... More first to expand the menu
3

Import the solution package

  1. Click Import solution in the top toolbar
  2. Click Browse and select RMAReturnsMonitor_1_0_0_2.zip
  3. Click Next — Power Apps validates the package (~10 seconds)
  4. Review the solution information shown (RMAReturnsMonitor v1.0.0.2)
  5. Click Next
4

Set up the connection

  1. You'll see a prompt asking for the Microsoft MCP Servers connection
  2. Click + New connection next to it
  3. A new tab opens — sign in with your Microsoft 365 account
  4. Once authenticated, close that tab and return to the import wizard
  5. Click the refresh icon next to the connection dropdown
  6. Select your newly created connection
Don't see "Microsoft MCP Servers"? Your tenant may not have premium connectors enabled. Contact your Power Platform admin or licensing team — see Section 7 for help.
5

Complete the import

  1. Click Next
  2. Click Import
  3. Wait approximately 1–3 minutes — Power Apps creates the 5 tables and installs the app
  4. You'll see "Solution imported successfully" when complete
6

Verify the install

  1. Click into the RMA Returns Monitor solution
  2. Confirm you see 6 components: 5 tables + 1 Canvas App
  3. Click Apps in the left filter
  4. You should see RMA Returns Monitor listed
Installation complete! Continue to Section 4 for first-run configuration.

4. First-run setup & configuration

Before users can effectively use the app, configure your plants and routing rules. This takes about 10–15 minutes.

Step 1 — Add your manufacturing plants

  1. In the solution, click into the Plant table
  2. Click + New row
  3. Fill in for each plant:
    • Name — e.g. "Detroit Manufacturing"
    • Region — North America / Latin America / Asia Pacific
    • Part Prefixes — comma-separated list of part-number prefixes this plant handles (e.g. "BR,BN")
    • Product Lines — comma-separated list of product line names
    • Auto Credit Threshold — dollar amount above which manager approval is required (e.g. 5000)
  4. Click Save
  5. Repeat for each plant in your network

Step 2 — Add routing rules

  1. In the solution, click into the Routing Rule table
  2. Click + New row
  3. Fill in:
    • Rule Name — descriptive name (e.g. "BR-prefix → Detroit")
    • Rule Type — Part Prefix / Customer Region / Product Line
    • Match Value — the value to match (e.g. "BR" for the rule above)
    • Assigned Plant — pick the plant this rule routes to
    • Priority — lower numbers evaluate first (e.g. 10, 20, 30)
    • Is Active — Yes
  4. Click Save
  5. Add as many rules as needed to cover your routing scenarios

Step 3 — Share the app with your team

  1. In the solution, locate the RMA Returns Monitor app
  2. Click the ... menu → Share
  3. Add users or security groups
  4. Pick the appropriate permission level:
    • Co-owner — for admins (rare; usually just you)
    • User — for everyday RMA processors
  5. Optional: Send an email notification with the app link
Security note: When sharing the app, also configure data permissions on the underlying tables. Each user needs at least Read/Write on the 5 RMA tables. Use Dataverse security roles to manage this.

5. Using the app

Launching the app

  1. Go to https://make.powerapps.com → Apps
  2. Click RMA Returns Monitor
  3. The app opens in a new tab

Typical workflow

StageWhat happens
IntakeAn RMA request comes in (email, web form, or direct entry). User creates a new claim in the app.
TriageApp suggests a plant assignment based on routing rules. User confirms or overrides.
InvestigationUser adds Claim Notes documenting findings. Verifies warranty status.
DecisionUser sets Resolution (Credit Issued / Replacement / Repair / Denied). If credit exceeds plant threshold, an Approval Record is created.
ClosedUser updates Status to "Closed" with Closed Date.

6. Troubleshooting

IssueCauseSolution
Import fails with "missing publisher" Solution file corrupted in transit Re-download the .zip and try again
"Microsoft MCP Servers" connector not available Premium connectors not enabled in tenant Contact your Power Platform admin to verify licensing
App opens but tables show "no records" Empty tables on first install (expected) Add data via Section 4 setup, or import sample data
"You don't have permission to access this data" User missing Dataverse table permissions Assign appropriate security role with read/write on rma_* tables
Routing not assigning correctly Routing rules misconfigured or inactive Review the Routing Rule table — confirm Is Active = Yes and Priority order is correct
"License required" prompt when running app User missing Power Apps Premium license Assign per-user or per-app premium license to the user

7. Support & next steps

Need help?

For installation or configuration issues, contact your Microsoft account team. Reference this solution as RMAReturnsMonitor v1.0.0.2.

Recommended next steps

  1. Pilot the app with 2–3 users for 1–2 weeks to validate the workflow against your real RMA volume.
  2. Capture feedback on missing fields, additional rule types, or report needs — share in your follow-up call.
  3. Plan integration — common follow-on enhancements include:
    • Auto-create RMAs from inbound emails (Power Automate flow)
    • Customer-facing web form that posts directly to the RMA Claim table
    • Auto-triage AI agent that classifies eligibility and drafts replies
    • Status notification emails to customers as their claim progresses
    • Power BI dashboard for cycle-time and disposition reporting

About the design

This app was built collaboratively in Power Apps Studio (Vibe code mode) and uses 5 custom Dataverse tables specifically designed for return management — no Field Service or other prerequisites required. The schema is documented in the solution and can be extended by your team using the standard Power Platform tools.

You own this solution. Because it's distributed as unmanaged, your team can extend tables, modify the app, and add new features as your RMA process evolves. We recommend keeping the original rma_* table schemas intact and adding new columns or related tables as you need them.
